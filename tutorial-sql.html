SQL 上手指南

建表与索引、增删改查、连接与聚合、常见误区与练习题等。

基础语法

-- 建表
CREATE TABLE users(id INTEGER PRIMARY KEY, name TEXT NOT NULL, age INT);
-- 查询
SELECT name, age FROM users WHERE age >= 18 ORDER BY age DESC;
-- 连接
SELECT u.name, o.amount FROM users u JOIN orders o ON u.id=o.user_id;
-- 聚合
SELECT COUNT(*) AS cnt, AVG(age) AS avg_age FROM users;

常见坑

  • 索引：根据查询模式创建合适索引，避免全表扫描。
  • N+1：合理使用连接或聚合，避免重复请求。
  • 事务与隔离：关键操作使用事务，理解隔离级别。

练习题

  1. 设计一个“用户-订单”的数据库，并编写统计查询。
  2. 为常用查询添加索引并对比性能变化。
  3. 实现一条分页查询与总量统计的 SQL。

给完全新手：第一步怎么做

  1. 安装 SQLite 或使用 MySQL/PostgreSQL，确认客户端可连接。
  2. 创建数据库与表，插入若干行示例数据。
  3. 编写几条查询与更新语句，理解事务与回滚。
  4. 添加索引并比较查询性能。

项目实战（入门）

  1. 用户与订单：设计表结构与关联，编写统计查询。
  2. 分页与总量：实现分页查询与总量统计。
  3. 报表生成：导出为 CSV 或可视化图表。

进阶内容

  • 索引策略：覆盖索引、组合索引与选择性。
  • 查询优化：合理设计、避免隐式类型转换。
  • 事务：规划边界，减少锁争用。

更多练习

  1. 在电商场景设计订单、仓储与优先级相关的表与查询。
  2. 对关键查询进行索引与语法优化。
  3. 实现分区与物化视图，并对多维统计做聚合报表。

系统化学习路径

  1. 建模与语法：表与约束、连接与聚合、视图与存储过程。
  2. 索引与优化：规划策略、索引选择性、避免 N+1 与隐式转换。
  3. 事务与运维：隔离等级、锁与并发、监控与审计。

进阶项目

  1. 多维报表系统：分区与滚动、做聚合与缓存、按需生成报表。
  2. 权限与合规：变更记录、数据脱敏、权限模型与多租户。
  3. 数据管道：从 CSV/JSON 清洗与建模，ETL，对大数据查询接口。
